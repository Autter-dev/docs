<AccordionGroup>
  <Accordion title="Do you train AI on my data?">
    No. Your code is never used to train our models. We take security and privacy seriously — your source code is encrypted in transit and at rest, and is never stored beyond the duration of a review.
  </Accordion>

  <Accordion title="What programming languages do you support?">
    Autter is language-agnostic and supports all popular programming languages, including JavaScript, TypeScript, Python, Go, Ruby, Java, C#, Rust, Kotlin, and Swift.
  </Accordion>

  <Accordion title="Do you have an open source plan?">
    Yes. Autter is free for public repositories. Connect Autter to a public repository to get unlimited AI reviews.
  </Accordion>

  <Accordion title="How does Autter integrate with my workflow?">
    Autter integrates directly with GitHub and GitLab. Once connected, it automatically reviews every pull request and merge request — no extra setup or steps needed from your team.
  </Accordion>

  <Accordion title="What kinds of issues does Autter catch?">
    Autter catches bugs, security vulnerabilities, performance issues, code style violations, and logic errors. It also suggests improvements for readability and maintainability, going beyond what traditional linters can detect.
  </Accordion>

  <Accordion title="How fast are the reviews?">
    Most reviews complete within seconds of a pull request being opened. Autter runs in parallel with your CI pipeline so it never slows down your development workflow.
  </Accordion>

  <Accordion title="Can I customize the review rules?">
    Yes. Configure Autter to follow your team's coding standards, ignore specific file patterns, and focus on the areas that matter most to your codebase through a simple configuration file.
  </Accordion>

  <Accordion title="Does Autter replace human code reviews?">
    No. Autter is designed to augment your team, not replace it. It handles the repetitive and easy-to-miss issues so your reviewers can focus on architecture, design, and business logic.
  </Accordion>

  <Accordion title="How do you handle private and proprietary code?">
    Your code is processed in isolated, ephemeral environments. We never store your source code after a review is complete, and all data is encrypted end-to-end.
  </Accordion>
</AccordionGroup>
